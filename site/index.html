Quick Start

Paste this into your AI agent on each machine.

Read the README at https://github.com/EthanSK/agent-bridge and follow the setup instructions for this computer. Install agent-bridge, run the setup command, and install the skill for this agent. Do everything automatically — don’t ask me questions.
Prerequisite: Enable SSH on each machine first — macOS: System Settings → General → Sharing → toggle Remote Login ON · Linux: sudo systemctl enable --now sshd

Then photograph the pairing screen and send it to the agent on the other machine. That’s it — they can now talk to each other.

How it works

Three steps. Then it’s invisible.

  1. 1

    Setup

    Run agent-bridge setup on each computer. It generates SSH keys, creates a one-time pairing token, and displays a pairing screen showing your local IP, public IP, and connection details.

  2. 2

    Pair

    Photograph one machine’s pairing screen and send it to Claude on the other machine. Claude reads the image, extracts the connection details, verifies the pairing token, and completes the pairing automatically.

  3. 3

    Use

    “Run the tests on my MacBook” just works — from either machine. Both peers are equal. Add --agent to run an AI agent prompt on the remote machine.

Features

Everything you need for multi-machine AI agents.

Photo-based pairing

Photograph the pairing screen. Claude reads the image, extracts connection details, and pairs automatically. No typing IPs.

Bidirectional

No fixed roles. Both machines are equal peers — either one can run commands on the other. Pair once, communicate both ways.

Remote command execution

Run any shell command on paired machines. Deploy apps, check status, manage processes — all from either computer.

Agent-to-agent

Use --agent to run an AI prompt on the remote machine. Delegate work to Claude, Codex, Aider, or any agent on any paired peer.

SSH key auth

ED25519 keys, no passwords. Keys stored with restrictive permissions. One-time pairing codes. No cloud servers involved.

Zero dependencies

Pure bash. No runtimes, no package managers. Just bash, ssh, and ssh-keygen — built into every Mac and Linux box.

Internet tunnel

Add --internet to expose your machine via serveo.net — no port forwarding needed. Also works with Tailscale hostnames.

Multi-agent support

Ships with config files for Claude Code, Codex, Gemini CLI, and OpenClaw. Each agent automatically knows how to use the bridge.

Compatibility

Works with every major AI coding agent.

Claude Code

skills/bridge/skill.md — Copy to ~/.claude/skills/. Claude learns to use the bridge automatically.

Codex CLI

AGENTS.md — Auto-detected from repo root. No extra setup needed.

Gemini CLI

GEMINI.md — Auto-detected from repo root. No extra setup needed.

OpenClaw

skills/openclaw/SKILL.md — Copy to ~/.openclaw/workspace/skills/.

Aider & others

INSTRUCTIONS.md — Plain-English reference that works with any agent that reads files.
