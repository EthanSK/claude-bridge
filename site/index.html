How it works

Three steps. Then it’s invisible.

  1. 1

    Setup the target

    Run npx claude-bridge setup on the machine you want to control remotely. It enables SSH, generates keys, and displays a pairing screen with connection details and a QR code.

  2. 2

    Pair via photo

    Photograph the pairing screen and send it to the controller Claude (via Telegram, chat, or any channel). Claude reads the image, extracts the connection details, and saves the pairing. Or paste the manual command.

  3. 3

    Bridge your Claudes

    “Run the tests on my MacBook” just works. The controller SSH’s into the target, executes the command, and reports back. Add --claude to run a full Claude Code prompt on the remote machine.

Features

Everything you need for multi-machine Claude.

Photo-based pairing

Photograph the target’s setup screen. Claude reads the image, extracts connection details, and pairs automatically. No typing IPs.

Remote command execution

Run any shell command on paired machines. Deploy apps, check status, manage processes — all from your main workstation.

Claude-to-Claude

Use --claude to run a Claude Code prompt on the remote machine. Your controller Claude can delegate work to the target Claude.

SSH key auth

ED25519 keys, no passwords. Keys stored with restrictive permissions. One-time pairing codes. No cloud servers involved.

QR code pairing

The setup screen includes a QR code with all connection details encoded. Scan it or photograph it — either way, pairing takes seconds.

Claude Code skills

Ships with ready-made skills for both controller and target. Claude knows how to use the bridge without any extra prompting.

Tailscale ready

Works with Tailscale hostnames for cross-network connections. Your Claudes can talk to each other from anywhere.

Zero config

One command on each machine. No config files to edit, no ports to forward, no tokens to manage. It just works.
