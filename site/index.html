How it works

Three steps. Then it’s invisible.

  1. 1

    Set up each machine

    Run claude-bridge setup on each computer you want to bridge. It enables SSH, generates keys, and displays a pairing screen with connection details.

  2. 2

    Pair via photo

    Photograph one machine’s pairing screen and send it to the Claude on the other machine (via Telegram, chat, or any channel). Claude reads the image, extracts the connection details, and saves the pairing.

  3. 3

    Bridge your Claudes

    “Run the tests on my MacBook” just works — from either machine. Each peer can SSH into the other, execute commands, and report back. Add --claude to run a full Claude Code prompt on the remote machine.

Features

Everything you need for multi-machine Claude.

Photo-based pairing

Photograph the pairing screen. Claude reads the image, extracts connection details, and pairs automatically. No typing IPs.

Bidirectional

No fixed roles. Both machines are equal peers — either one can run commands on the other. Pair once, communicate both ways.

Remote command execution

Run any shell command on paired machines. Deploy apps, check status, manage processes — all from either computer.

Claude-to-Claude

Use --claude to run a Claude Code prompt on the remote machine. Your Claude can delegate work to the Claude on any paired peer.

SSH key auth

ED25519 keys, no passwords. Keys stored with restrictive permissions. One-time pairing codes. No cloud servers involved.

Zero dependencies

Pure bash. No Node.js, no Python, no package managers. Just bash, ssh, and ssh-keygen — built into every Mac and Linux box.

Tailscale ready

Works with Tailscale hostnames for cross-network connections. Your Claudes can talk to each other from anywhere.

Zero config

One command on each machine. No config files to edit, no ports to forward, no tokens to manage. It just works.
