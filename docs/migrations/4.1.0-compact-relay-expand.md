# 4.1.0 compact relay expand ids

## Context

OpenClaw Agent Bridge relay notices used to include a large `message:` preview in Telegram/user-facing receipts. Ethan asked for compact receipts instead: show a short expand id, keep the full bridge body in a local bounded store, and let agents expand it on demand.

This update bumps agent-bridge CLI/MCP metadata to 4.1.0 and openclaw-channel to 3.1.0. The OpenClaw channel now writes recent full inbound BridgeMessages to `~/.agent-bridge/relay-expand/` and relay receipts show `expand id: NN` plus `agent-bridge relay-expand NN`.

## Instructions for the agent receiving this update

1. Run the standard repo update/build flow for this machine (`git pull`, then `npm install`/`npm run build` in `mcp-server` if your install requires it).
2. Confirm the local CLI reports `agent-bridge 4.1.0` with `agent-bridge version`.
3. If this machine runs OpenClaw with the native `openclaw-channel/` plugin, confirm `openclaw-channel/package.json` is `3.1.0` in the loaded repo path.
4. Do **not** paste full bridge bodies into Telegram/user-facing relay notices by default. If Ethan asks to “expand Agent Bridge relay message NN”, run `agent-bridge relay-expand NN` on the same machine that produced the notice and send the retrieved content subject to normal privacy/channel rules.
5. A live OpenClaw gateway/plugin may need its normal safe reload/restart before already-running sessions pick up the new channel code. Do not kill active user-facing sessions without explicit operator approval; if you cannot safely reload, report that a reload is required.

## Verification

- `agent-bridge version` prints `agent-bridge 4.1.0`.
- `node --test openclaw-channel/test/relay-notice.test.js openclaw-channel/test/relay-expand-store.test.js openclaw-channel/test/relay-expand-cli.test.js` passes from the repo root.
- New OpenClaw relay notices contain `expand id:` and do not contain a `message:` preview.

## Rollback

Revert to the previous repo commit and reload the affected harness/plugin. Existing `~/.agent-bridge/relay-expand/` files are local cached message copies; leave them in place or remove them only if the operator explicitly asks for cleanup.
